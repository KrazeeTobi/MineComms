The Mod API involves rewriting most of Minecraft code (which is what they've been doing for a while now) to make it properly moddable and easy to update and extend in the future. Forge is a giant mess and will only increase the workload each update. The original author of the Minecraft Coder Pack (Searge) now works at Mojang and is also working on the Mod API. Whenever the Mod API is finished (including in-game mod selecting/installing Mojang has talked about) it'll make all other APIs (Forge etc) redundant. Given the Mod API should in theory be more extendable and way more stable and less resource intensive, mod authors would need to update to it instead of using older things such as Forge.
