EDIT: I think I just need a deeper explanation on the config files in group manager, like global groups, then specific world groups and users etc. I guess half of it, sometimes it works and sometimes not :s[/QUOTE] put all your perms in globalgroups.yml, typically if you have essentials the one you want to use as the main one is "g:essentials_regular" or "g:essentials_default" but you can rename this to anything you want as long as you keep the g:. Then, go to config.yml so you can set up the mirrors (if you have multiworld of course). Mine is set up like this: Towny: Creative: - users - groups VG: - users - groups Nether: - users - groups The first world, Towny, is what all other worlds will copy their permissions from. This should be the default spawning world. All the worlds below it are the ones that will copy the permissions. Do this so you don't have to update like 4 permission folders every time. Next, go into the worlds folder and then into your default world folder. Open groups and give the groups you want the permissions from globalgroups.yml, should look a bit like this: Default: default: true permissions: [] inheritance: - g:essentials_default Permissions should be set up fine like that.
