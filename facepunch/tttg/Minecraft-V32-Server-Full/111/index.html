I love how you guys are doing calculations based off of the absolutely worst case that notch posted. Think about it for a second...How many of you are going to join a server and just run in a totally opposite direction then everyone else? never see anyone again, just run and keep running until you fill up the memory on the server. That's where Notch's number comes from. In reality, players are going to hang out near eachother, building, mining, and groups of players will occasionally explore. There isn't a number attached to that, since it's incredibly variable. You can probably get away with 30 players on 1gb of ram, provided they act normal, and don't go off and generate new territory every second with noone else around. Also, you have to factor in that RAM wont be the only bottleneck. There is a point where no matter how much memory you throw at the application, you just can't run that many players. I'm willing to bet that when mobs get added, that point will be around 30 players. This game is not counterstrike with a clean cut environment, it generates new terrain and you can modify the terrain. You have to send new map data to clients and keep track of it...something that no games on the market to day have to do.
