On the left a fresh batch is shot up in the sky[/url] [editline]8th November 2010[/editline] [QUOTE=frznbxy;25940885]That actually spawned another question in me: How hard is it to make mods in Minecraft? Or better, how do you make mods? There's quite some things I'd like to see in minecraft, but while notch is being busy making texture pack support (I mean, come on :/), I'd try doing some of them myself. It's not really open-source or anything, right? So how would you change something like that sand-falling-upwards thing?[/QUOTE] It depends, really. When Java is made into an executable file (or a jar, whatever) it is converted into machine-language and your human eyes can't read it. You'll need to run the minecraft.jar through a decompiler, edit the java classes, recompile and test it. I recommend using the de-obfuscator on the Minecraft forums, otherwise all classes and names look like "eb" instead of "CraftingRecipe". Depending on what you want to make, it doesn't have to be hard. changing what block spawns what, or even adding some items isn't too hard. It is when those items have to do something that it becomes tough. So basically, if you can read javacode, you can give it a try. If you can write javacode, then it should be possible for you to make some simple mods. There are some tutorials on the minecraft forums. At the moment, I'm trying to make a block that nullifies falldamage, but I haven't found where the falldamage is calculated yet. There are, like, hundreds of classes in here!
