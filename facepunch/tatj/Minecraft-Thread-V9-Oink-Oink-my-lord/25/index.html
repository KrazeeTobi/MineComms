Today's Update [quote=Notch]Level generator: The level generator is about twice as fast now. This can be optimized further.. And I kind of changed it in the process, heh. :D The big fix here was to make it pull entire chunks of raw data from the noise function rather than just one sample at a time. Unfortunately, splitting it out into an asynchronous thing caused massive problems (monsters falling down into nothing, then getting stuck in ground when the land suddenly appeared, rendering glitches, even more uneven framerate, problems with saving/loading maps while it was generating), so I ended up ditching that. Saving to disk is much less aggressive now, but this means that the game pauses for longer when quitting or switching map, as it has to save out more unsaved chunks. I can do a lot of tweaks to this, such as saving entities and level data separately. Monster spawner: The maximum number of monsters and animals is the same, but they should spawn much slower now, both increasing performance on slow computers and tweaking the balance a bit. The huge groups of monsters will get tweaked as well. Some of them fit in groups, others should be solo, I will tweak the count per monster type. Also, I fixed a couple of major bugs in the spawning code which let them spawn in bright areas.. Sorry about that. Miscellaneous Fixes: Planted trees now use the correct tree type. The pathfinder no longer leads mobs over cliffs in certain areas of the map (negative coordinates) Finally, a warning: Don&#8217;t delete the level you&#8217;re currently playing. It won&#8217;t explode your computer or anything, but it will produce strange results.[/quote] [quote]I will fix this as soon as possible (thursday?), but: If you close the browser window while playing a level, the browser can crash. This is because Minecraft goes &#8220;OH CRAP, I&#8217;M GETTING SHUT DOWN!? I&#8217;D BETTER HURRY UP AND SAVE EVERYTHING TO DISK&#8221;, and if that takes too long, the browser gets upset. To prevent this, always select &#8220;quit game&#8221; from the menu before closing the browser. I will make the game only save the nearby area when quitting the game that way. This means you can lose changes to the level if you close your browser window, but at least it won&#8217;t crash the browser.[/quote]
