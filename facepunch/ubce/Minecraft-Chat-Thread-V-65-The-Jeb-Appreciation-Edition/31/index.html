For those asking about permissions and groups, the new permissions included in Essentials v2.1, v2.2 and v2.3 contains a feature that allows you to add users and groups safely in-game. To user these you either have to be an Operator or be in a group with the correct permissions node to edit the files. [code]************************* List of Commands I Know Exist ************************* USER-SPECIFIC COMMANDS ---------- /manuadd <username> <groupname> - Manually ADD a user to a specified group (e.g. /manuadd notch Admin) /manuaddp <username> <permissionnode> - Add a permission to an individual(e.g. /manuaddp notch essentials.warp) /manuaddv <username> <variable> <value> - Add a variable to an individual(e.g. /manuaddv notch prefix UPDATEPLZ) /manudelp <username> <permissionnode> - Delete a permission from an individual(e.g. /manudelp notch consistent.updates) /manudelv <username> <variable> - Delete a variable from an individual(e.g. /manudelv notch codingschedule) (VARIABLES SHOULDN'T BE DELETED OR ELSE YOU MIGHT SCREW SOMETHING UP WITH THE USER!!) GROUP-SPECIFIC COMMANDS ---------- /mangadd <groupname> - Create a whole new group (e.g. /mangadd Super) /mangaddp <groupname> <permissionsnode> - Add a permissions to a group(e.g. /mangaddp Super essentials.kick) /mangaddv <groupname> <variable> <value> - Add a variable to a group(e.g. /manuaddv Super prefix &5) /mangdelp <groupname> <permissionnode> - Delete a permission from a group(e.g. /manudelp default essentials.home) /mangdelv <groupname> <variable> - Delete a variable from a groupl(e.g. /manudelv Super prefix) (VARIABLES SHOULDN'T BE DELETED OR ELSE YOU MIGHT SCREW SOMETHING UP WITH THE GROUP!!)[/code]These are probably the only permissions commands you'll need to know, apart from Manudelv and Mangdelv because you shouldn't really delete variables from users/groups.
