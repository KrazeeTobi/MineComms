For the total number of loaded chunks across all players, divide that count by 256 and multiply by 70 to get the total number of hostile mobs that can spawn throughout the server. So each player should be receiving 8 chunk radius (?) from the server, which is ~289 chunks per player, minus players' chunks who are grouped up in the same area. I can't find any information saying the actual spawners spawn based on the hostile mob count, so they probably do not. So for example, you have 80 players on the server. There's a few groups of people together working so effectively there are 60 players spread across the server. That's 60 * 289 = 17340 chunks. Divide by 256 = 18.3 then multiply by the constant for hostile mobs (70) = and you get a mob cap of 1281 mobs. Now if you have only 6 people at mob spawners with ~200 mobs currently spawned... that leaves 81 mobs to be spawned across the remaining chunks for all 80 players. It should not be a surprise if no mobs spawn at all in your area in this case, even at night; especially slimes, which only spawn 1/10th as often as other hostile mobs.
